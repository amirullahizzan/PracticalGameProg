ゲームプログラム応用　課題　２次元配列マップの操作
----------------------------------------------------------------------

提出期限：１時間後（できているところまでの提出で構いません、そのあと解説していきます）
提出場所：teams->ゲームプログラム応用->授業（本日）->ファイル->提出

プロジェクトフォルダの名前を【 プロジェクト名_番号_自分の名前 】のように変更して提出してください。
（例えば【 ゲームプログラム応用_課題00_00000000_○○○○○ 】みたいな感じで）

以下の変更を実装することで【 こういう感じで.exe 】のような挙動になるようにしてください。


課題１：マップ全体の描画位置を画面の真ん中あたりにする

	今回のマップはサンプルの物よりも画像サイズが半分になっていますのでパネル１個１個は小さく描画されています。
	パネルサイズの定数も半分の大きさにしています。
	↓こんな感じで
	#define PANEL_SIZE	25

	ですので、マップ番号を入れる２次元配列の変数やルート変数の要素数は少し多めにしています。
	↓こんな感じで
	#define MAP_W		20
	#define MAP_H		13

	そんなマップですが、現状の描画は画面左上を基準にしているため、文字列の描画と被って見えにくいです。

	そのために、マップをどのくらいの所から描画するかの定数を用意しました。

	↓こんな感じの物
	//	マップ全体はこの基準の場所から描画をおこなう
	#define BASE_X	150
	#define BASE_Y	105

	この値を使ってずらした座標でマップや文字列やＮＰＣなどの描画をおこないます。

	あと、クリックしたときのパネルの場所もそのずれ分に合わせる必要があります。


課題２：クリックしたときのルート作成

	現状は、クリックしたときに開始ポイントの設定だけをおこなっており、
	ルートを広げていく処理はスペースキーを押した時におこなっています。

	今回のプロジェクトではクリックしたときに、１０マス分のルートを作成します。
	（スペースキーを１０回押した結果と同じにします・・・スペースキーを押したかどうかの判定は使いません）

	さらに、その１０マス分のルートが見やすくなるようにエリア画像（ area.bmp ）の描画をします。

	area.bmp：他のパネル画像と同じ大きさ（２５ｘ２５）です。

	この画像はすでに読み込んでいます。
	↓こんな感じで
	area_image = LoadGraph( "data/area.bmp" );

	「こういう感じで.exe」では、このエリア描画を半透明の設定で描画しています。
	↓こんな感じで
	SetDrawBlendMode( DX_BLENDMODE_ALPHA, 100 );	//	半透明設定
	DrawGraph( x, y, area_image, TRUE );			//	エリア画像の描画
	SetDrawBlendMode( DX_BLENDMODE_NOBLEND, 0 );	//	半透明設定を戻す


ちなみに
	画面サイズ　　　　　　　　　　　８００ｘ４５０
	ＮＰＣ画像（ npc.bmp ）　　　　　 ２５ｘ　２５
	パネル画像（ panel.bmp ）　　　　 ２５ｘ　２５
	黄色い枠の四角画像（ mark.bmp ）　２５ｘ　２５
	ルートエリア画像（ area.bmp ）　　２５ｘ　２５
です。


各処理に関して、既に宣言してある変数や関数だけでは処理が実装できない場合、
各自で変数や関数を追加した上で実装をおこなってください。
座標や色・スピードなど細かい数値の指定は特にないので、各自でいい感じの数値を設定することで
【 こういう感じで.exe 】のような動きになるように実装してください。
